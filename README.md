# 📊 SQL for Data Analysis

데이터 분석 업무에서 필요한 SQL 역량을 체계적으로 훈련하고 요구사항을 쿼리로 구조화하는 능력을 증명하기 위한 레포지토리입니다.  
단순 문제 풀이 저장이 아니라, 문제를 데이터 조회/집계/관계 해석 관점으로 재구성하여 기록합니다.

---

## 🎯 What this repository proves

이 레포지토리는 다음 역량을 보여주기 위해 만들어졌습니다.

- 비즈니스 요구사항을 SQL 로직으로 변환하는 능력
- 관계형 데이터 구조 이해 및 JOIN 설계 능력
- 집계 기준 정의 및 데이터 해석 능력
- 반복 사용 가능한 조회 패턴 정리 능력
- 계층 구조 및 복잡 쿼리 설계 능력

---

## 🧠 Problem Solving Approach

모든 문제는 아래 과정으로 해결합니다.

1. 데이터가 의미하는 기준 정의
2. 필요한 데이터 집합 설계
3. 조건과 집계 기준 분리
4. 단계별 쿼리 구성
5. 재사용 가능한 패턴으로 일반화

정답을 맞추는 것보다 **다른 데이터에서도 동일하게 동작하는 쿼리 설계**를 목표로 합니다.

---

## 📁 Repository Structure

SQL 개념 학습 순서가 아닌 실제 분석 작업 흐름 기준으로 정리했습니다.

```
week01_select_where/   : 필요한 데이터 정의 (조건 필터링)
week02_join/           : 데이터 관계 연결 (테이블 결합)
week03_sum_max_min/    : 지표 생성 (집계 함수 SUM, MAX, MIN, COUNT)
week04_is_null/        : 데이터 품질 처리 (NULL 핸들링)
week05_group_by/       : 기준별 요약 (GROUP BY, HAVING)
week06_string_date/    : 전처리 및 변환 (문자열 함수, 날짜 함수)
```

---

## 🏷 Query Classification System

문제는 난이도가 아니라 **사용된 데이터 처리 방식** 기준으로 분류합니다.

| 코드 | 의미 |
|------|------|
| `S` | 조건 기반 조회 |
| `J` | 관계 결합 |
| `G` | 집계 및 지표 계산 |
| `W` | 분석 함수 |
| `R` | 계층 구조 처리 |

**Example**

```
S-001 | 기본 조건 필터링
S-002 | 문자열 패턴 검색
J-001 | 관계 기반 개수 집계
G-001 | 평균 지표 산출
R-001 | 계층 Leaf 노드 계산
```

→ 특정 문제 풀이가 아니라 **필요한 쿼리 패턴을 바로 찾기 위한 구조**입니다.

## 💡 Why this matters

실무 SQL은 단순 조회가 아니라 **데이터 정의 → 관계 해석 → 지표 생성 → 검증** 과정입니다.  
이 레포지토리는 문제 풀이 기록이 아니라 데이터 분석 업무에서 반복되는 **쿼리 설계 패턴을 축적**하기 위해 작성되었습니다.
